What is EgoLang?

EgoLang is a strongly-typed programming language, designed to give developers more precise control over variable visibility and mutability. With a clear structure and secure data access, EgoLang makes code more reliable and predictable.

Developed to prioritize security and clarity, EgoLang includes modern programming concepts with a simple and direct syntax. Every statement in the language is explicit about scope, modification capability, and data type, which eliminates many of the ambiguities found in other languages.

Main Features

Visibility Control

Clearly define the access scope for each variable and function with explicit visibility modifiers.

Explicit Mutability

Precisely specify if a variable can be modified after declaration, increasing code security.

Strong Typing

Strict type system to ensure consistency and avoid common errors during program execution.

Error Detection

Precise identification of code problems during compilation, before they become runtime bugs.

JIT Compilation

Uses Just-In-Time compilation to transform EgoLang code into optimized Python bytecode.

Clear Structure

Clean and explicit syntax that makes code more readable and maintenance simpler.

Basic Syntax

Comments

EgoLang supports single-line and multi-line comments:

// This is a single-line comment

/* This is a
   multi-line
   comment */

Variables

Variables in EgoLang must be declared with visibility, mutability, type, and name:

visibility mutability type name = value;

For example:

public mutable int counter = 0;
private immutable string name = "Ego";
protected const bool isEnabled = true;

Visibility Modifiers

  • public: Accessible from anywhere
  • private: Limited to the current scope
  • protected: Limited to the current scope and child scopes

Mutability Modifiers

  • mutable: Can be changed after declaration
  • immutable: Cannot be changed after declaration
  • const: Cannot be changed (similar to immutable)

Data Types

  • int: Integer values
  • float: Floating-point values
  • string: Text values
  • bool: Boolean values (true/false)
  • list: List of values
  • dict: Dictionary/map of key-value pairs

Control Structures

Conditionals (if)

if (condition) {
    // code to run if the condition is true
} else {
    // code to run if the condition is false
}

Loops

For Loops:

for (initialization; condition; update) {
    // code to run in the loop
}

While Loops:

while (condition) {
    // code to run while the condition is true
}

Functions

Functions are declared with visibility, the function keyword, name, parameters, and body:

visibility function name(parameters) {
    // function body
}

For example:

public function calculateArea(int width, int height) {
    public mutable int area = width * height;
    return area;
}

Printing Values

print(value);

Practical Examples

Basic Variable Usage

public mutable int x = 10;
public immutable string greeting = "Hello";
print(x);
print(greeting);

Using Conditions

public mutable int age = 18;

if (age >= 18) {
    print("Adult");
} else {
    print("Minor");
}

Loop Example

Using While:

public mutable int i = 0;
while (i < 5) {
    print(i);
    i = i + 1;
}

Using For:

for (public mutable int j = 0; j < 5; j = j + 1) {
    print(j);
}

Function Example

public function add(int a, int b) {
    return a + b;
}

public mutable int result = add(5, 3);
print(result);

Error Handling

The EgoLang compiler will generate errors for:

  1. Attempts to modify an immutable variable
  2. Use of undefined variables
  3. Invalid visibility or mutability modifiers
  4. Invalid data types
  5. Syntax errors in control structures or function declarations

Error messages will be included as comments in the generated Python code.

Compilation Process

The EgoJITCompiler processes Ego code in these steps:

  1. Cleans the source code (removes comments, normalizes whitespace)
  2. Divides the code into logical blocks
  3. Translates each block to equivalent Python code
  4. Tracks variables and their properties in scopes
  5. Compiles the generated Python code to bytecode
Execution Environment: Compiled Ego programs run in the Python runtime environment, but maintain Ego's variable access and mutability rules through the generated code.

Language Limitations

  • No support for classes or object-oriented programming
  • Limited data type operations
  • No built-in exception handling
  • Limited standard library

Using the Compiler

  1. Go to the src/Ego_compiler file and scroll to the end of the code.
  2. Inside the sample_code instance, write your code:

Example:

sample_code = """

public mutable int x = 10;
print(x);

"""
  1. In the function:
compiler.compile(sample_code, 'output.ego')

Set the output filename by replacing 'output.ego' with the name you want for the compiled file.

Example:

compiler.compile(sample_code, 'my_file.ego')
